스터디 디렉토리와 http-server 설정

바탕화면에 study 폴더를 생성해 파일들을 한 폴더로 정리했고, web server를 사용해 web page를 띄워볼 것이다

  1. 먼저node.js를 다운 받는다
  2. GitHub - http-server에서 다운 받은 후 cmd를 열어 install --global http-server를 입력한다

    그러면 cmd 어느 폴더에서든 http-server를 실행할 수 있다

    폴더는 C:\Users\Klete_장경환\node_modules에 저장된다
  3. 설치 후http-server -a 127.0.0.1 -p 8080이라고 입력하면 server가 실행된다

    -a → 사용할주소

    -p → 열고 싶은port 번호

    폴더의 경로를 추가하면 그 폴더의 index.html이 실행된다 →http-server Desktop/study -a 127.0.0.1 -p 8080



  4. URL창에주소:port 번호를 입력한다

    만약 index.html이 없다면 해당 폴더에서 html파일을 선택할 수 있다

HTML

HTML은 web page를 제작하기 위한 표준마크업 언어다

web page의 구조를 나타내며,element와 element에 기능을 제공하는attribute가 있다



    HTML page의 구조
  • HTML 유형의 문서는선언과 함께 시작해야 한다 →!DOCTYPE html
  • HTML의 문서는 html태그 사이에 작성해야 한다
  • HTML의 문서에서 보이는 부분은body태그사이에 작성한다
  • head태그는 web page에서 보이지 않고,메타 데이터를 표현하는 영역이다.


마우스 오른쪽 클릭을 하고페이지 소스 보기를 선택하면 문서의 소스 코드를 볼 수 있다

혹은검사, 요소 검사를 선택하면 요소 검사 창이 열리고 html이나 css를 편집할 수 있다

HTML element

element는시작 태그, 컨텐츠, 닫는 태그로 구성되어 있다


element는중첩 사용이 가능하다

닫는 태그를 작성하지 않으면 이상한 결과가 나올 수 있어닫는 태그는 필수로 작성해야 한다

link태그나 img태그처럼 닫는 태그를 작성하지 않는element가 있다


태그들은 대소문자를 구분하진 않지만 되도록소문자로 적는 것이 좋다

HTML attribute

HTML attribute는 element에기능을 제공한다

이름 설명
href 태그에하이퍼링크 기능을 제공한다
src URL을 명시한다. img태그, script태그 등경로가 필요한 element에 사용한다
alt 사용자가이미지를 볼 수 없거나 스크린 리더를 사용하는 경우대체 정보를 제공한다
sytle element의style을 지정해주고, 값으로css 문법이 들어간다. 하나의element에만 적용된다
lang element의 언어를 정의한다
title element에 대한추가 정보를 지정한다. element 위에 마우스를 올리면 정보가 표시된다.

HTML 시멘틱 태그

다른 태그들과 달리컨테이너의 역할을 하며의미만 부여한다

이전의 html은div태그를 사용하여 자유롭게 class나 id명을 정의했었다

하지만 개발자들의접근성이나검색 엔진 최적화를 위해 시멘틱 태그를 사용하고 있다

검색 엔진은시멘틱 태그를 통해 html을더 쉽게 분석할 수 있다

이름 설명
header 머리글 지정, 로고나 제목 태그, nav태그도 포함한다. head태그와 다르다
nav 다른 사이트의 문서로 연결하는 링크의 모음으로 주로메뉴에 사용된다
section 주제별 콘텐츠 영역으로그룹핑하기 위해 사용한다. section 안에header와 footer가 있을 수 있다
article 콘텐츠의실제 내용을 포함하는 영역이다
aside 본문 이외의 내용으로 사이드에 위치하여광고나 링크로 구성되어 있다
footer 하단에 위치하여 회사 소개, 저작권 등을 포함하는 영역이다
